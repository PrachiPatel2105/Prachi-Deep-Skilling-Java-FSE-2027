WHAT IS MOCKITO?
================
- Framework for creating mock objects
- Used for unit testing
- Isolates code under test
- Controls external dependencies
- Makes tests fast and reliable

WHY USE MOCKITO?
================
1. Test code with external dependencies
2. Avoid calling real APIs in tests
3. Control what dependencies return
4. Test error scenarios easily
5. Make tests fast (no network calls)
6. Isolate unit under test

MOCKITO CONCEPTS:

1. MOCK (mock())
   - Creates a fake object
   - Behaves like real object
   - Can be controlled in test
   
   Example:
   ExternalApi mockApi = mock(ExternalApi.class);

2. STUB (when().thenReturn())
   - Configure mock's behavior
   - Specify what method returns
   - Set return values for specific inputs
   
   Example:
   when(mockApi.getData()).thenReturn("Mock Data");

3. VERIFY (verify())
   - Confirm method was called
   - Check if mock was used
   - Verify interactions
   
   Example:
   verify(mockApi).getData();

4. TIMES (verify(..., times(n)))
   - Verify call count
   - Check exact number of calls
   
   Example:
   verify(mockApi, times(3)).getData();

5. NEVER (never())
   - Verify method NOT called
   - Ensure method wasn't invoked
   
   Example:
   verify(mockApi, never()).getData();

MOCKITO METHODS USED:

mock()
- Creates mock object
- Usage: mock(Class.class)
- Returns mock instance

when()
- Starts stubbing configuration
- Usage: when(mock.method())
- Chainable with thenReturn()

thenReturn()
- Specify return value
- Usage: when(...).thenReturn(value)
- Called after when()

verify()
- Verify method was called
- Usage: verify(mock).method()
- Can be combined with times()

times()
- Verify exact call count
- Usage: verify(mock, times(n))
- n = number of times

never()
- Verify method NOT called
- Usage: verify(mock, never())
- Equivalent to times(0)

COMPARISON: WITHOUT vs WITH MOCKS

WITHOUT MOCKS:
@Test
public void test() {
    ExternalApi api = new RealApi(); // Real API!
    MyService service = new MyService(api);
    String result = service.fetchData();
    // Problems:
    // - Slow (network call)
    // - Network dependent
    // - Can't test errors easily
}

WITH MOCKS:
@Test
public void test() {
    ExternalApi mockApi = mock(ExternalApi.class); // Fake!
    when(mockApi.getData()).thenReturn("Mock");
    MyService service = new MyService(mockApi);
    String result = service.fetchData();
    // Benefits:
    // - Fast (no network)
    // - Always reliable
    // - Easy error testing
    // - Fully controlled
}
