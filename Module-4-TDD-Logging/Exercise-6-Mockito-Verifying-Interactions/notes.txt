WHAT IS VERIFYING INTERACTIONS?
================================
- Confirming that mock methods were called
- Checking arguments passed to methods
- Verifying order of method calls
- Ensuring no unexpected calls made

WHY VERIFY INTERACTIONS?
========================
1. Ensure correct data is passed
2. Verify workflow sequence
3. Test integration between components
4. Validate API usage

MOCKITO VERIFICATION CONCEPTS:

1. VERIFY WITH ARGUMENTS
   verify(mock).method(argument);
   - Checks method called with exact argument
   - Verifies data passed correctly
   
   Example:
   verify(service).updateBalance("ACC123", 1000.0);

2. ARGUMENT CAPTOR
   ArgumentCaptor.forClass(Type.class)
   - Captures actual argument value
   - Inspect captured value in detail
   - Verify complex objects
   
   Example:
   ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
   verify(service).method(captor.capture());
   String actual = captor.getValue();

3. INORDER VERIFICATION
   InOrder inOrder = inOrder(mock);
   inOrder.verify(mock).methodA();
   inOrder.verify(mock).methodB();
   - Verify execution order
   - Important for workflows
   
   Example:
   InOrder inOrder = inOrder(service);
   inOrder.verify(service).log(...);
   inOrder.verify(service).process(...);
   inOrder.verify(service).send(...);

4. VERIFY NO MORE INTERACTIONS
   verifyNoMoreInteractions(mock);
   - Ensure only expected calls made
   - Prevent unexpected side effects
   - Comprehensive verification
   
   Example:
   verify(service).expectedMethod();
   verifyNoMoreInteractions(service);

5. VERIFY WITH ARGUMENT MATCHERS
   verify(mock).method(anyString());
   - Match any argument
   - More flexible verification
   - Use when exact value irrelevant
   
   Examples:
   - anyString() - any string
   - anyInt() - any integer
   - any() - any object
   - eq(value) - exact match

VERIFICATION METHODS LEARNED:

verify(mock).method(arg)
- Basic argument verification
- Exact argument matching

verify(mock, times(n)).method(arg)
- Verify call count with argument
- Called exactly n times

verify(mock, never()).method(arg)
- Verify never called with argument
- Opposite of normal verify

InOrder inOrder = inOrder(mock)
inOrder.verify(mock).methodA()
inOrder.verify(mock).methodB()
- Verify method call sequence
- Critical for workflows
- Ensures order is maintained

ArgumentCaptor.forClass(Type.class)
- Capture argument passed
- Extract actual value
- Assert on captured value

verifyNoMoreInteractions(mock)
- Verify no unexpected calls

- Comprehensive validation
