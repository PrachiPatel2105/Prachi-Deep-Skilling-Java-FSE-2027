KEY CONCEPTS LEARNED:

RECURSION BASICS:
- Function calls itself to solve subproblems
- Must have base case (when to stop)
- Must have recursive case (move toward base case)
- Call stack depth increases with each call
- Risk: Stack overflow on deep recursion

FINANCIAL FORECASTING FORMULA:
FutureValue = PresentValue × (1 + GrowthRate)^Periods

Example:
- Initial: $1,000
- Rate: 8% per year
- Time: 5 years
- Future: $1,000 × (1.08)^5 = $1,469.33

FOUR APPROACHES TESTED:

1. NAIVE RECURSIVE:
   Time: O(n) - linear
   Space: O(n) - call stack
   Pros: Simple, shows recursion clearly
   Cons: Stack overhead, slower than iteration

2. MEMOIZATION:
   Time: O(n) - improved from O(2^n) for Fibonacci
   Space: O(n) - array for memoization
   Pros: Faster for problems with overlapping subproblems
   Cons: Still recursive, still stack overhead

3. DYNAMIC PROGRAMMING:
   Time: O(n) - linear
   Space: O(n) - array
   Pros: Iterative (no stack), predictable
   Cons: Less intuitive than recursion

4. MATHEMATICAL FORMULA:
   Time: O(1) - constant
   Space: O(1) - no extra space
   Pros: Instant, elegant, production-ready
   Cons: Only works when formula exists

FIBONACCI COMPARISON:
Without memoization:
- fib(10): 177 calls

With memoization:

- fib(30): 61 calls

- fib(50): 101 calls (works instantly!)

Speed improvement: 50,000x faster for fib(30)!

FIBONACCI LESSON:
This exercise shows why naive recursion is bad:
- 2^n growth is exponential
- Each level of recursion doubles calls
- Memoization reduces to O(n)
- Better yet: use iteration
- Best: use closed-form formula if it exists

FINANCIAL FORECASTING LESSON:

- Formula is perfect for this problem
- No need for recursion at all
- O(1) solution beats O(n) every time
- This teaches when NOT to use recursion
