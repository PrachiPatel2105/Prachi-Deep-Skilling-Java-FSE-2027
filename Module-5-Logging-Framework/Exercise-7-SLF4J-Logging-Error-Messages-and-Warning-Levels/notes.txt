WHAT IS SLF4J?
==============
- Simple Logging Facade for Java
- Abstraction layer for logging
- Allows switching implementations
- Works with Logback, Log4j, etc.

WHY USE SLF4J?
==============
1. Decoupling - Application not tied to specific logging library
2. Flexibility - Can switch implementations
3. Performance - Lazy string formatting
4. Simplicity - Easy API

LOGGING LEVELS (Severity):
===========================

1. TRACE (Most detailed)
   - Trace execution flow
   - Method entry/exit
   - Usually disabled in production
   
2. DEBUG
   - Debug information
   - Variable values
   - Development/troubleshooting
   
3. INFO (Default)
   - Informational messages
   - Business events
   - Application lifecycle
   - Enabled in production
   
4. WARN
   - Warning messages
   - Unusual but recoverable
   - Potential problems
   - Example: Deprecated usage
   
5. ERROR (Most severe in use)
   - Error messages
   - Failures
   - Exceptions
   - Application can continue
   
6. FATAL/OFF
   - Fatal (some implementations)
   - OFF = no logging

EXAMPLE USE CASES:

TRACE:
- logger.trace("Entering method with args: {}", args);
- logger.trace("Processing loop iteration: {}", i);

DEBUG:
- logger.debug("Processing order: {}", orderId);
- logger.debug("Account balance: {}", balance);
- logger.debug("Method execution time: {}ms", duration);

INFO:
- logger.info("User logged in: {}", username);
- logger.info("Payment processed successfully");
- logger.info("Application started");
- logger.info("Database connection established");

WARN:
- logger.warn("Low memory: {}MB available", freeMemory);
- logger.warn("Deprecated method used");
- logger.warn("Invalid input, using default");
- logger.warn("Connection timeout, retrying");

ERROR:
- logger.error("Database connection failed", exception);
- logger.error("Payment processing failed: {}", error);
- logger.error("File not found: {}", filename);
- logger.error("Invalid user data: {}", errorDetails, exception);

SLF4J IMPLEMENTATION:

Facade (Interface):
- org.slf4j.Logger
- org.slf4j.LoggerFactory

Implementation (Logback):
- ch.qos.logback.core.ConsoleAppender
- ch.qos.logback.core.FileAppender

Configuration:
- logback.xml in src/main/resources

USING SLF4J:

1. Create Logger
   private static final Logger logger = 
       LoggerFactory.getLogger(ClassName.class);

2. Log Messages
   logger.info("User: {}", username);
   logger.error("Error: {}", message, exception);

3. Parameterized Messages (Efficient)
   - GOOD: logger.info("User: {}", user);
   - BAD: logger.info("User: " + user);
   - Avoids string concatenation overhead

LOGBACK CONFIGURATION (logback.xml):

Pattern Symbols:
%d - Date/time
%thread - Thread name
%-5level - Log level (padded)
%logger - Logger name
%msg - Message
%n - Newline

Pattern Example:
%d{yyyy-MM-dd HH:mm:ss} %-5level %logger - %msg%n

Output:
2026-07-02 14:30:45 INFO com.bank.MyClass - User logged in

Appenders:
- ConsoleAppender - Output to console
- FileAppender - Output to file
- RollingFileAppender - Output to rotating files

Root Logger Configuration:
<root level="INFO">
  <appender-ref ref="CONSOLE"/>
  <appender-ref ref="FILE"/>
</root>

Package-specific Levels:
<logger name="com.bank" level="DEBUG"/>
<logger name="org.springframework" level="WARN"/>

LOGGING BEST PRACTICES:

1. Use Correct Levels
   ✓ TRACE for method flow
   ✓ DEBUG for debug info
   ✓ INFO for business events
   ✓ WARN for unusual situations
   ✓ ERROR for failures

2. Use Parameterized Messages
   ✓ logger.info("User {}", name);
   ✗ logger.info("User " + name);

3. Include Context
   ✓ logger.error("Failed for {}: {}", id, msg);
   ✗ logger.error("Failed");

4. Log Exceptions Properly
   ✓ logger.error("Error", exception);
   ✗ logger.error(exception.toString());

5. Don't Log Sensitive Data
   ✓ logger.info("Auth for user: {}", username);
   ✗ logger.info("Password: {}", password);

6. Use Static Final Logger
   ✓ private static final Logger logger = ...
   ✗ private Logger logger = ...

COMMON PATTERNS:

Pattern 1: Operation Logging
logger.info("Operation started: {}", operation);
try {
    // do operation
    logger.info("Operation completed: {}", operation);
} catch (Exception e) {
    logger.error("Operation failed: {}", operation, e);
}

Pattern 2: Method Logging
logger.debug("Entering method with params: {}", params);
try {
    // method logic
    logger.debug("Exiting method with result: {}", result);
} catch (Exception e) {
    logger.error("Method failed", e);
}

Pattern 3: Validation Logging
if (!isValid(input)) {
    logger.warn("Invalid input: {}", input);
    logger.error("Processing failed due to invalid input");
}

Pattern 4: State Change Logging
logger.debug("Current state: {}", currentState);
currentState = newState;
logger.info("State changed to: {}", currentState);
