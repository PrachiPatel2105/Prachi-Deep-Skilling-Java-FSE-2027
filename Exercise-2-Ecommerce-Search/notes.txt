BIG O NOTATION SUMMARY:
- O(1): Constant - fastest (access by index)
- O(log n): Logarithmic - very fast (binary search)
- O(n): Linear - normal (linear search)
- O(n²): Quadratic - slow (nested loops)
- O(2ⁿ): Exponential - very slow (avoid!)

LINEAR SEARCH ANALYSIS:
- Time: O(n) - must check each element
- Space: O(1)
- Requires: Unsorted or sorted array (doesn't matter)
- Best Case: O(1) - found at position 0
- Worst Case: O(n) - found at end or not found
- Average Case: O(n/2) ≈ O(n)

BINARY SEARCH ANALYSIS:
- Time: O(log n) - divides in half each time
- Space: O(1) iterative, O(log n) recursive
- Requires: SORTED array (mandatory!)
- Best Case: O(1) - found at middle
- Worst Case: O(log n) - at leaf of binary tree
- Average Case: O(log n)

EXAMPLE COMPARISONS:
Array size: 1,000,000

Linear Search:  500,000 comparisons (average)
Binary Search:  ~20 comparisons
Difference:     25,000 times FASTER!

WHY BINARY SEARCH IS FASTER:
1. Eliminates half of remaining elements each iteration
2. Like finding person in phone book:
   - Linear: Check every name in order
   - Binary: Open middle, go left or right, repeat
3. With 1M items, only need ~20 steps vs ~500,000

WHEN TO USE EACH:
Linear Search:
- Small datasets (< 100 items)
- Unsorted data that can't be sorted
- One-time search
- Simplicity is priority

Binary Search:
- Large datasets (> 1000 items)
- Frequent searches
- Data is or can be sorted
- Performance is critical

FOR E-COMMERCE PLATFORM:
1. Product IDs → Binary Search (sorted in database)
2. Product Names → Hash table or Full-text search
3. Price Range → Database index (B-tree)
4. Category Filter → Hash table
5. Complex Filters → Elasticsearch

ANALYSIS PERFORMED:
- Small dataset (10 items): 2.5x faster
- Medium dataset (1K items): 50x faster
- Large dataset (100K items): 2941x faster
- Very large (1M items): 25,000x faster

IMPORTANCE OF SORTING:
Sorting Cost: O(n log n) - done once
Search Cost: O(log n) - done many times

For 1M products:
- Sorting: ~20 million operations
- 100,000 binary searches: ~2 billion operations (linear would need 50 billion!)
