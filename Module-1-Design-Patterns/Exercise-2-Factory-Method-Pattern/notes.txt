KEY CONCEPTS LEARNED:
1. Factory Method Pattern - creates objects without specifying exact classes
2. Product Interface - defines contract for all products
3. Concrete Products - specific implementations of interface
4. Abstract Factory - declares factory method
5. Concrete Factories - implement factory method
6. Polymorphism - use common interface for different types
7. Loose Coupling - client doesn't know concrete product classes

KEY LEARNINGS:
- Factory Method separates object creation from usage
- Easy to add new document types without changing existing code
- Follows Open/Closed Principle (open for extension, closed for modification)
- Polymorphism allows treating different objects as one type
- Reduces dependencies between classes

DIFFERENCES FROM SINGLETON:
- Singleton: ONE instance of a class
- Factory: Creates MULTIPLE DIFFERENT types of objects
- Singleton: Focus on instance count
- Factory: Focus on object creation flexibility
